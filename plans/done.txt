1. S3 Image Storage (was local disk, now AWS S3)
Problem: The original code saved uploaded images to the container's local disk (multer.diskStorage). On a server, every redeploy wipes the disk — all images would disappear.

What we changed:

apps/api/src/config/s3.js (new file)

Creates an AWS S3 client using S3_REGION + AWS_ACCESS_KEY_ID + AWS_SECRET_ACCESS_KEY
Exports uploadBuffer(key, buffer, mimeType) → uploads bytes to S3, returns public URL
Exports deleteObject(key) → deletes from S3 (fire-and-forget, never throws)
apps/api/src/routes/imageRoutes.js (modified)

Switched multer.diskStorage → multer.memoryStorage() (file stays in RAM, never touches disk)
Magic-byte security check (detects real file type from bytes, blocks spoofed extensions) now runs on req.file.buffer instead of a disk path
Attaches req.file.detectedExt for the controller to use
apps/api/src/controllers/imageController.js (modified)

Added STORAGE_DRIVER switch: if s3 → call s3.uploadBuffer(), store S3 URL + storageType:'s3'; if disk → write to local disk as before
deleteImage now deletes from S3 when storageType==='s3', disk otherwise
Local dev unchanged — STORAGE_DRIVER defaults to disk
**`apps/api/src/controllers/bulkImportController.js

continue
apps/api/src/controllers/bulkImportController.js (modified)

Added saveImage(buffer, originalFilename, mimetype) helper → writes to S3 or disk based on STORAGE_DRIVER
Added deleteStoredImage(filename) helper → deletes from S3 or disk
Rollback on transaction failure now cleans up S3 objects (not just disk files)
cleanupFiles() is now async to handle S3 deletes
apps/api/src/app.js (modified)

/uploads static file serving only mounts when STORAGE_DRIVER !== 's3' — in production S3 serves images directly, no need for the server to serve them
apps/api/src/config/env.js (modified)

Added: STORAGE_DRIVER, S3_BUCKET, S3_REGION, S3_PUBLIC_URL, AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY
Validation: if STORAGE_DRIVER=s3, boots fails immediately if S3_BUCKET or S3_REGION are missing
apps/api/package.json (modified)

Added @aws-sdk/client-s3 v3 dependency (306 MB final Docker image)
